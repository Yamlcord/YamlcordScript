# 🛠️ Contributing Guide

Ευχαριστούμε που θέλετε να συμβάλετε στο project!  
Ακολουθούν οι οδηγίες για να το κάνετε με τον καλύτερο δυνατό τρόπο.

---

## 📌 1. Πριν ξεκινήσετε
- Ελέγξτε τα **ανοιχτά Issues** ώστε να αποφύγουμε διπλές αναφορές.
- Διαβάστε το **README** για να καταλάβετε τη λειτουργία και τη δομή του project.
- Αν έχετε απορία, ανοίξτε ένα Issue τύπου **question** πριν ξεκινήσετε ανάπτυξη.

---

## 🐛 2. Αναφορά Προβλημάτων (Issues)
> **Σημείωση:** Όλα τα Issues πρέπει να γράφονται στα **αγγλικά**, ώστε όλοι οι συνεισφέροντες να μπορούν να τα κατανοήσουν.

Όταν δημιουργείτε νέο Issue:
- Χρησιμοποιήστε καθαρό και περιγραφικό τίτλο.
- Περιγράψτε λεπτομερώς το πρόβλημα.
- Προσθέστε βήματα αναπαραγωγής, αν υπάρχουν.
- Επισυνάψτε logs, snippets ή screenshots όπου βοηθούν.

---

## 🔧 3. Συνεισφορά Κώδικα (Pull Requests)
1. Κάντε **fork** το repository.  
2. Δημιουργήστε νέο branch με περιγραφικό όνομα, π.χ.:  
   - `fix/comment-scanner-bug`
   - `feature/add-division-operator`
3. Κάντε τις αλλαγές σας και βεβαιωθείτε ότι:
   - Ο κώδικας ακολουθεί το style του project.
   - Οι αλλαγές είναι καθαρές, μικρές και modular.
   - Όλες οι δοκιμές (αν υπάρχουν) περνούν χωρίς σφάλματα.
4. Ανοίξτε Pull Request και περιγράψτε:
   - Τι ακριβώς αλλάξατε
   - Το σκεπτικό πίσω από την αλλαγή
   - Οποιαδήποτε χρήσιμη πληροφορία για όσους κάνουν review

---

## 📝 4. Code Style
- Γράφουμε ευανάγνωστο, καθαρό κώδικα.
- Αποφεύγουμε μεγάλα και δυσανάγνωστα functions.
- Προτιμούμε περιγραφικά ονόματα μεταβλητών/μεθόδων.
- Διατηρούμε συνεπή μορφοποίηση σύμφωνα με τα standards του project.

---

## 🤝 5. Συνεργασία & Σεβασμός
- Φιλικός και επαγγελματικός τόνος σε όλα τα σχόλια.
- Καλωσορίζουμε συνεισφορές από όλους, ανεξαρτήτως εμπειρίας.
- Spam issues ή PRs διαγράφονται άμεσα για καθαρό ιστορικό.

---

## 📬 6. Απορίες;
Μπορείτε πάντα να ανοίξετε Issue με label **question** και θα προσπαθήσουμε να βοηθήσουμε όσο το δυνατόν πιο άμεσα.

---

Ευχαριστούμε που στηρίζετε το project! 🚀
